# Technology Stack

## Recommended Technologies
- **Frontend**: HTML5, CSS3, JavaScript (vanilla or React)
- **State Management**: Local storage for game progress
- **Animation**: CSS animations or JavaScript libraries
- **Design**: Mobile-first responsive approach

## Project Structure
- Static web application (no backend required)
- JSON-based game data configuration
- Image assets for scenes and characters
- Single-page application architecture

## Key Technical Requirements
- Random city generation for starting location
- Clue randomization within difficulty tiers
- Route tracking to prevent revisiting cities
- Score/time tracking system
- Win/loss condition handling
- Restart functionality

## Asset Management

## Common Commands
Since this is a static web project, typical commands would be:
- `python -m http.server 8000` - Local development server
- `npx serve .` - Alternative local server (if using npm)
- No build process required for vanilla implementation
- For React version: standard `npm start`, `npm run build`

## Data Structure
- Centralized game configuration in `assets/data/game_data.json`
- Multilingual support (Spanish primary, English elements)
- Structured city data with informants, clues, and responses
